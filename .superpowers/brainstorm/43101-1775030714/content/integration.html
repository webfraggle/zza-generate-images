Wie wird der Node-Editor integriert?

Der bestehende Editor hat: Dateiliste | CodeMirror (YAML) | JSON-Input + Vorschau

A

Ersetzt den YAML-Editor vollständig

Der Node-Editor ist der Editor. Kein CodeMirror mehr. YAML wird nur noch im Hintergrund generiert und gespeichert. Einfacheres UI, keine Dopplung.

Pro

  • Klare Oberfläche
  • Kein Sync-Problem

Con

  • Fortgeschrittene verlieren direkten YAML-Zugriff
B

Tab-Umschalter: Node-Editor ↔ YAML

Zwei Tabs im Code-Bereich: „Nodes" und „YAML". Wechsel synchronisiert bidirektional. Beide Modi bleiben vollständig nutzbar.

Pro

  • Maximale Flexibilität
  • Fortgeschrittene können weiter YAML tippen

Con

  • Bidirektionaler Sync ist komplex
  • YAML → Nodes kann bei komplexen Templates fehlschlagen
C

Eigene Seite: /edit/{token}/nodes

Der Node-Editor ist eine separate Route neben dem bestehenden Editor. Kein Sync nötig — wer Nodes nutzt, speichert von dort. Wer YAML will, nutzt /edit/{token}.

Pro

  • Keine Interferenz mit bestehendem Editor
  • Einfachste Implementierung

Con

  • Zwei Einstiegspunkte können verwirren
