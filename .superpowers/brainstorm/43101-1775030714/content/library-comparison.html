Rete.js vs Litegraph.js

Beide sind Vanilla-JS-kompatibel und framework-unabhängig

A — Rete.js v2
Lizenz MIT
Größe ~40 KB (Core), modular
GitHub Stars ~10k ⭐
Letztes Release 2024, aktiv gepflegt
Vanilla JS Ja — aber API ist klassen-basiert, Plugin-System
Nodes rendern HTML/DOM — volle CSS-Kontrolle
Verbindungen SVG-Bezier, typisierte Sockets

Pro

  • Moderne API, TypeScript-first
  • Nodes sind echtes HTML → einfach zu stylen
  • Gute Doku + Beispiele
  • Plugin für Minimap, History, etc.

Con

  • v2 bricht mit v1 — wenig Community-Content für v2
  • Etwas verbose Initialisierung
  • Kein CDN-Bundle → braucht Build-Step oder ESM-Import
D — Litegraph.js
Lizenz MIT
Größe ~120 KB (alles in einer Datei)
GitHub Stars ~5k ⭐ (bekannt durch ComfyUI)
Letztes Release 2023, wenig aktiv
Vanilla JS Ja — eine JS-Datei, kein Build nötig
Nodes rendern Canvas 2D — kein HTML, Custom-Drawing
Verbindungen Canvas-Bezier, typisierte Slots

Pro

  • Eine Datei, per Script-Tag einbindbar
  • Kein Build-Step nötig
  • Bewährt in ComfyUI (riesige Community)
  • Sehr performant (Canvas, kein DOM)

Con

  • Canvas-Rendering → Nodes stylen ist aufwändig
  • Kaum aktive Wartung seit 2023
  • API ist älter, weniger intuitiv
  • Custom-Widgets für Dropdowns etc. komplex

Meine Empfehlung: Rete.js

Für dieses Projekt sind die Nodes HTML-DOM — das macht Formularfelder (Farb-Picker, Dropdowns, Text-Inputs) in den Nodes trivial. Litegraph zeichnet alles auf Canvas, was für interaktive Inputs in Nodes viel Aufwand bedeutet. Rete.js passt besser zum bestehenden Vanilla-JS + CSS Ansatz. Der fehlende Build-Step lässt sich mit esm.sh lösen (wie CodeMirror bereits eingebunden ist).
